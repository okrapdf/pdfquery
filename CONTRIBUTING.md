# Contributing to pdfQuery

Thank you for your interest in contributing to pdfQuery! We welcome contributions from the community to help make this tool better for everyone.

## How to Contribute

### Reporting Bugs
- Search existing issues to see if the bug has already been reported.
- If not, create a new issue with a clear title and description.
- Include steps to reproduce the bug and any relevant document examples if possible.

### Feature Requests
- Check existing issues to see if the feature has already been requested.
- If not, open a new issue describing the feature and why it would be useful.

### Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or bug fix: `git checkout -b feature-name`.
3. Make your changes and ensure they follow the project's style.
4. Add tests for your changes if applicable.
5. Run the existing tests to ensure no regressions: `npm test`.
6. Commit your changes: `git commit -m 'Add some feature'`.
7. Push to the branch: `git push origin feature-name`.
8. Open a Pull Request on the main repository.

## Development Setup

```bash
# Install dependencies
npm install

# Build the project
npm run build

```
